# 设计规范：学术机构穿透与同名作者幻觉阻断架构
*日期：2026-04-06*

## 1. 核心问题诉求 (Problem Statement)
当底层元数据 API（Semantic Scholar 和 Crossref）发生“机构（Affiliation）字段真空”时，纯人名的大模型泛搜索（尤其在遇到拼音高度重合的华人姓名时）会极大概率引发**张冠李戴的幻觉**（如本案中由于缺乏机构定界，St. Jude 实验室的 Yaqiu Wang 被错误联想搜索成了 UCSF 的 Yaqiu Wang）。必须基于多层漏斗式的后备方案，严防死守不带机构的“裸搜”。

## 2. 改进架构与流水线设计 (Architecture & Data Flow)

### 2.1 基础元数据管线拓宽 (`doi_resolver.py` & `author_extractor.py`)
*   **提取侧**：在查阅 Crossref 记录时，遍历增加并捕获 `author[x]['ORCID']` 字段。
*   **通信侧**：将 `ORCID` URL（若存在）持久化录入作者提取列表内，并在全工作流透传至末端的方法签命中（即将 `find_email_for_paper` 函数接口拓宽，能够接收 `orcid` 及 `pi_org` 变量）。

### 2.2 防幻觉三级漏斗机制 (`email_finder.py` 核心重构)

整个系统搜邮箱之前的判断将按顺序依次经过如下三个漏斗过滤器：

#### 过滤漏斗一：ORCID 绝对绑定
*   若待搜作者含有明确的 `ORCID`：在所有向千问发出的底层 Prompt 前，强行拼入前缀：“学术识别极高安全警告：该学者的唯一标识符为 `[ORCID URL]`。你在全网查找邮箱和进行确权验证时，只允许基于该信息溯源对应的机构和身份信息，严格杜绝同名泛搜。”

#### 过滤漏斗二：时序乱序与学界惯例继承（归属大老板算法）
*   **时序锚定**：对提取循环进行控制——**必定先搜索并锚定“通讯作者”的机构信息**。因为通讯作者保留全量机构、自带 ORCID 及全网主页曝光的概率通常极高。
*   **血统继承**：在着手搜索“第一作者（一作）”且发现该作者没有任何机构时。自动调取刚才锁定的大老板的机构 `corr_org`，作为子级上下文传递给大模型，下达指令：“该作者暂无元数据独立机构。但依据师承规律，它当前系该篇论文第一作者，大概率与通讯作者同处于一家研究机构或 PI Lab。请务必结合 `{corr_org}` 的信息去排查该同名一作的实际邮箱。如果不在本列表及合作机构内，应当退回未找到”。

#### 过滤漏斗三：盲搜熔断与 HTML 底层爬刮（休克疗法）
*   若遇到了两人机构都不全且都没有 ORCID 等极其反常的极限场域。
*   严禁大模型触发 `enable_search=True` 的纯名称联想裸搜。
*   触发预配置好的 `_scrape_paper_page()` 函数获取目标出版商官网对应的超大体积原生 HTML 富文本。
*   调用专用的小型 LLM 阅读任务：`smart_recover_affiliation(名字, HTML全文文本)`，单纯读取出目标原本的机构，填补上空缺后，再正式执行邮箱查找流水线即可万无一失。

## 3. 防断言错误边界控制 (Error Handling)
在最终整合阶段，`_qwen_search_roundX` 里的判定准则加上红线：若在检索最终轮中，仍然无法通过任何上下文反证出找到的邮箱是确实隶属于这篇论文提及的研究人员，系统将一律强制判定为 `"未找到"` ，抛弃其作为候选目标，以根绝对后续推送错误引流。

## 4. 验证测试基准 (Testing)
使用基准问题 `10.1016/j.cell.2025.11.002`。
*   **验证 1**：控制台输出：成功爬取识别到通讯作者 T. Kanneganti 相关的 ORCID 。
*   **验证 2**：控制台输出：第一作者 Yaqiu Wang 输出并触发继承策略，在搜索时附加通讯作者所在的 St. Jude 医院约束。
*   **验证 3**：最终解析结果，Yaqiu Wang 第一作者不应输出任何有关 `UCSF` 的错误定界数据；邮箱将要么为精准捕获，要么安全退化为 `"未找到"` 输出。
